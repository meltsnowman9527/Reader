# Reader
用于个人搜集到的漫画、小说、图片以及视频管理，解决搜集癖管理资源难得痛点，用AI写出来的小玩意，Spaghetti code，仅限个人用途，不允许任何商业用途，不喜勿喷。
一个运行在本地的个人多媒体管理与阅读工具，支持漫画（ZIP/CBZ）、图片、视频、小说四种内容类型，通过浏览器访问，无需安装客户端。

## 功能特性

### 内容类型
- **漫画**：支持 ZIP / CBZ 压缩包上传，自动解析章节，逐页或卷轴翻阅
- **图片**：单图上传与管理，支持相册式翻阅（滑动切换）
- **视频**：视频文件上传，支持系列/集数管理，封面由用户手动截帧设置
- **小说**：TXT 文本上传，支持章节识别与阅读进度记录

### 阅读体验
- 漫画/图片双模式：单页模式（左右滑动）+ 卷轴模式（上下滚动）
- 单页模式支持相册式滑动预览（左右拖动可预览相邻页）
- 视频播放器自动根据分辨率适配窗口大小
- 阅读进度自动保存，下次打开继续上次位置

### 管理功能
- 标题、标签、作者、收藏分类编辑
- 漫画系列归档（多部合并为一个系列）
- 视频系列与集数管理
- 批量导出（打包下载）
- 重复内容检测
- 标签页、收藏分类快速筛选
- 关键词搜索 + 排序

### 移动端适配
- 响应式布局，手机浏览器可用
- 长按卡片弹出编辑按钮
- 底部安全区适配（iPhone 底部横条）

## 技术栈

| 层级 | 技术 |
|------|------|
| 服务端 | Node.js + Express |
| 图片处理 | sharp（自动生成缩略图） |
| 压缩包解析 | adm-zip / yauzl |
| 前端 | 原生 HTML + CSS + JavaScript（无框架） |
| 数据存储 | 本地 JSON 文件 |

## 快速开始

### 前置条件

- Node.js 18+

### 安装

```bash
git clone https://github.com/你的用户名/manga-reader.git
cd manga-reader
npm install
```

### 配置（可选）

在项目根目录创建 `.env` 文件：

```env
PORT=3000
READER_LOGIN_USER=admin
READER_LOGIN_PASS=your_password
```

不配置时默认端口 `3000`，使用内置默认账号。

### 启动

```bash
npm start
```

或直接双击 `启动阅读库.bat`（Windows）。

启动后在浏览器访问：`http://localhost:3000`

## 目录结构

```
manga-reader/
├── server.js              # 主服务入口
├── routes/
│   └── mediaRoutes.js     # 图片/视频 API 路由
├── public/
│   ├── index.html         # 主界面
│   ├── login.html         # 登录页
│   ├── js/
│   │   ├── app.js         # 主逻辑（漫画/小说/收藏/系列）
│   │   └── media.js       # 媒体逻辑（图片/视频查看器）
│   └── css/
│       └── style.css      # 样式
├── data/                  # JSON 数据库（自动生成）
│   ├── mangas.json
│   ├── novels.json
│   ├── images.json
│   └── videos.json
├── uploads/               # 上传文件（自动生成）
│   ├── images/
│   ├── image-thumbs/      # 图片缩略图缓存
│   └── videos/
└── novels/                # 小说 TXT 文件（自动生成）
```

## 注意事项

- `uploads/`、`novels/`、`data/` 目录均由程序自动创建，无需手动建立
- 数据存储在本地 JSON 文件中，无需数据库服务
- 上传的文件保存在 `uploads/` 目录，**请勿将该目录提交到 Git**（`.gitignore` 已排除）
